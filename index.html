What is non-backtracking?
Non-backtracking NonBT

Do not traverse the same edge in a row.

Non-backtracking realizes efficient propagation of data, information, etc.

example: Alon et al. proved non-backtracking random walk converges faster than traditional ones RW.

  1. 小蔵正輝，“複雑ネットワーク解析における非バックトラック，”電子情報通信学会誌，vol.105，no.1，pp.27-32，Jan. 2022．
  2. N. Alon, I. Benjamini, E. Lubetzky, and S. Sodin, "Non-backtracking random walks mix faster," Communications in Contemporary Mathematics, vol.9, no.04, pp.585-603, May 2007.

System model & problem definition

\(N\): Number of devices (connected via a network)

\(\Value{i}{0}\): initial value of device \(i\)

Devices communicate with their adjacent devices \(\Neighbors{i}\) to achieve consensus. \[\Value{i}{t+1}\leftarrow f(\{\Value{j}{t}\mid j\in\Neighbors{i}\})\]

Goal: calculate average \(\Value{i}{t} \rightarrow \frac{1}{N} \sum_{i=1}^{N} \Value{i}{0}\)

Basic consensus algorithm
Each device updates its value depending on the differences from its neighbors' values.
Scalar form: \(\Value{i}{t+1} \leftarrow \Value{i}{t} + \SharingRate \sum_{j\in\Neighbors{i}} (\Value{j}{t} - \Value{i}{t})\)
Vector form: \(\VecVals{t+1} \leftarrow \VecVals{t} - \SharingRate \Laplacian \VecVals{t}\)

\(\Laplacian\): Graph Laplacian

\(\Laplacian_{ij} = \begin{cases} |\Neighbors{i}| & \text{if } i=j \\ -1 & \text{else if } j \in \Neighbors{i} \\ 0 & \text{otherwise.}\\ \end{cases}\)

Convergency: \(\VecVals{t} = (\IdentityMat - \SharingRate \Laplacian)^t \VecVals{0} \rightarrow \frac{1}{{\NumDevices}} \Ones \VecVals{0}\)
\[\Laplacian = \begin{bmatrix} |\Neighbors{1}| & -1 & 0 \\ -1 & |\Neighbors{2}| & -1 \\ 0 & -1 & |\Neighbors{3}| \\ \end{bmatrix}\]
example of Laplacian
Propagating flows in networks
Flow $\Flow{\edge{i}{j}}{t}(\coloneqq\Value{i}{t}-\Value{j}{t})$: the amount of values moving from device \(i\) to device \(j\).
Creating directed graph $\DiGraph$ to analyze flow
Incidence matrix \(\IncidenceMat\) of \(\DiGraph\):
\[\IncidenceMat_{i,\edge{j}{k}} = \begin{cases} 1 & \text{if } i=j \\ -1 & \text{else if } i=k \\ 0 & \text{otherwise.} \\ \end{cases}\]
Rewrite updating functions:
\[\VecVals{t+1} \leftarrow \VecVals{t} - \SharingRate \Laplacian \VecVals{t}\]
⇓
\[\begin{align} \VecFlows{t} &\leftarrow \IncidenceMat^\Trans \VecVals{t} \nonumber \\ \VecVals{t+1} &\leftarrow \VecVals{t} - \frac{1}{2} \SharingRate \IncidenceMat \VecFlows{t} \nonumber \end{align}\]
$\Laplacian = \frac{1}{2} \IncidenceMat \IncidenceMat^\Trans$
Backtracks in basic consensus algorithm
Step $t-1$:
Flow $\Flow{\edge{i}{j}}{t-1}=\Value{i}{t-1}-\Value{j}{t-1}$ moves from $\#i$ to $\#j$.
Step $t$:
Flow $\Flow{\edge{j}{i}}{t}=\Value{j}{t}-\Value{i}{t}$ includes the previous flow flow $\Flow{\edge{i}{j}}{t-1}$ returning from $\#j$ to $\#i$
Proposed idea
suppress backtracks when updating devices' values
Suppressing backtracks
Conventional:
\[\begin{align} \VecFlows{t} &\leftarrow \IncidenceMat^\Trans \VecVals{t} \nonumber \\ \VecVals{t+1} &\leftarrow \VecVals{t} - \frac{1}{2} \SharingRate \IncidenceMat \VecFlows{t} \nonumber \end{align}\]
Proposed:
\[\begin{align} \VecFlows{t} &\leftarrow \IncidenceMat^\Trans \VecVals{t} + 2\CompRate\VecFlows{t-1}\nonumber \\ \VecVals{t+1} &\leftarrow \VecVals{t} - \frac{1}{2} \SharingRate \IncidenceMat \VecFlows{t} \nonumber \end{align}\]
Add a term $2 \CompRate \VecFlows{t-1}$ to suppress backtracking flow.
Implementation
Proposed flows can be calculated as follows:
\[ \begin{align} \VecFlows{t} &= \IncidenceMat^\Trans \VecVals{t} + 2\CompRate (\IncidenceMat^\Trans \VecVals{t-1} + 2\CompRate\VecFlows{t-2}) \nonumber \\ &= \sum_{\tau=0}^{t}(2\beta)^\tau \IncidenceMat^\Trans \VecVals{t-\tau} \nonumber \end{align} \]
Values $\VecVals{t}$ are updated as follows:
Vector form:
$\VecVals{t+1} \leftarrow \VecVals{t} - \SharingRate \Laplacian \sum_{\tau=0}^{t}(2\beta)^\tau \VecVals{t-\tau}$
Scalar form:
$\Value{i}{t+1} \leftarrow \Value{i}{t} + \SharingRate \sum_{j\in\Neighbors{i}} (\AvgValue{j}{t} - \AvgValue{i}{t})$
where $\AvgValue{i}{t} \coloneqq \sum_{\tau=0}^{t}(2\beta)^\tau \Value{i}{t-\tau}$
Conventional:
$\VecVals{t+1} \leftarrow \VecVals{t} - \SharingRate \Laplacian \VecVals{t}$
Values can be calculated without the flow $\Flow{\edge{i}{j}}{t}$ on each edge.

Non-backtracking consensus algorithm
$\VecUpdates{t}$: the amount of updates
\[\VecUpdates{t} \leftarrow - \SharingRate \Laplacian \VecVals{t} + 2\CompRate \VecUpdates{t-1}\]

Evaluation: Convergence speed of various topologies
Convergence time:
$\displaystyle{\min_t \{ \max_i \{|\Value{i}{t} - \IdealVal|\} \} < 10^{-10}}$
where $\IdealVal = \frac{1}{N}\sum_i^{N} \Value{i}{0}$
Num. devices: $N=100$
Initial values: $\Value{i}{0} \in [0, 10]$ (uniformaly random)
Topologies:
2D lattice, lattice ring, Watts Strogatz model WS, Barabási Albert model BA
  1. D.J. Watts and S.H. Strogatz, "Collective dynamics of 'small-world' networks," Nature, vol.393, no.6684, pp.440-442, June 1998.
  2. A.-L. Barabási and R. Albert, "Emergence of scaling in random networks," Science, vol.286, no.5439, pp.509-512, Oct. 1999.
2D lattice
  • Convergence speed increases with $\CompRate (< 0.5)$.
  • When $\CompRate \ge 0.5$, values diverge to $\infty$.
2D lattice ring
  • Convergence property is similar to that of 2D lattice.
Watts-Strogatz model
  • Small-world model (short average path length & high clustering)
  • Similar convergence property to lattice topologies
Barabasi-Albert model
  • Scale free model (Few nodes have high degree.)
  • Similar convergence property to lattice topologies
Summary
  • A non-backtracking consensus algorithm is proposed for sensor networks.
  • The proposed algorithm converges faster than conventional ones by surpressing the backtracking flows.
  • It can be implemented without additional costs of storing large data.
  • A similarity can be seen between the proposed algorithm and the momentum.
